# Open-Source & Copyleft
[Open-source](open-source) is a classification of software based on licensing.

Software which is not open-source is called closed-source, or [proprietary software](proprietary-software).

This **topic** is **strongly related** to, but **not exclusively** part of the [Free Software Movement](free-software-movement).

(w) Software with viewable source code but does not allow modification or redistribution is not considered open-source software, rather it is considered [proprietary software](proprietary-software).

**Open-source** projects often **struggle** with **funding development** due to being **non-commerical**. Many **projects** ask for **donations** in order to **fund development**.

**Open-source** projects rarely include so-called '**anti-features**' such as **advertisements**, **personal data collection**, or **user profiling**. This is because someone can just '**fork**' the **software** into their **own version** and **remove those features** and **distribute it** to others **anyway**.

(e) Popular applications such as [VLC](vlc), [Firefox](firefox), [Blender](blender), and [GIMP](gimp) are open-source software.

**Open-source** projects often (but **don't always**) allow users to **contribute changes** to the **main codebase**. This is **often** done through a [pull request](pull-request) (aka **merge request**) system, where users can **submit their changes** to the **project maintainers** for **review** and **potential inclusion** in the **main codebase**.

(e) The most contributed to open-source project is the [Linux](linux) kernel, which has had over 30,000 contributors and over 1.4 million [commits](commit) (contributions). You can browse a mirror of the project [here](https://github.com/torvalds/linux).

## Rationale
Some software developers **prefer** to allow users to **view**, **modify** and **redistribute** their software without their **permission**, but often with **conditions**. Some developers do this because they are supporters of the [Free Software Movement](free-software-movement), while others do it for **practical** or **business** reasons.

### Viral / Sharealike
So-called **viral** or **sharealike** licenses require that **if you modify** the software **and distribute it**, you **must** distribute your **modified version** under the **same license**. This can **act** as a **mechanism** to **spread the license** to other **software** which **want to use** a [Library](libraries) licensed under a **viral license**.

Many **developers** avoid using software **with viral licenses**, because they **do not want** to be **forced to release** their own software **under the same license**.
